Smart Pulse Oximeter

An IoT device for real-time remote health monitoring.

The Problem

Continuous monitoring of vital signs is crucial for managing chronic health conditions and for post-operative care. Traditional hospital monitoring can be restrictive, and at-home spot-checks don't provide a continuous data stream for analysis.

My Solution

I engineered a wearable Smart Pulse Oximeter using an ESP32 microcontroller. The device integrates a MAX30102 sensor to measure heart rate and blood oxygen saturation (SpO2), along with a DS18B20 sensor for body temperature. The data is processed in real-time and can be displayed on a local OLED screen and simultaneously transmitted to a cloud platform for remote monitoring by caregivers or medical professionals. This enables early detection of health anomalies and long-term trend analysis.

Challenges & Lessons Learned

The biggest challenge was accurately calibrating the bio-sensors and implementing algorithms to filter out motion artifacts from the readings. I also learned about the complexities of IoT connectivity and ensuring data security during transmission. This project was an excellent fusion of my skills in biomedical instrumentation, embedded systems, and cloud integration.

Technology Stack

  • ESP32 (C/C++)
  • MAX30102 Sensor
  • DS18B20 Sensor
  • IoT Cloud Platform

Key Features

  • Real-time SpO2 & Heart Rate monitoring.
  • Continuous body temperature tracking.
  • Local display and remote data sync.
  • Compact, wearable form factor.
